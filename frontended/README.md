# Smart Crop Advisor — Frontend Application

This is the Single Page Application (SPA) frontend for the **Smart Crop Advisor** system, built with React.js. It provides farmers and agricultural users with a responsive, multilingual dashboard to obtain AI-driven recommendations.

---

## 🚀 How to Run the Frontend

### 📋 Prerequisites
Ensure you have **Node.js** and **npm** installed on your system:
* **Node.js**: v18 or higher recommended ([Download Node.js](https://nodejs.org/))
* **npm**: Installed automatically with Node.js

### ⚙️ Setup Steps

1. **Navigate to the frontend directory**:
   Open a terminal in the root of the project, then run:
   ```bash
   cd frontended
   ```

2. **Install Node.js dependencies**:
   Installs React, React Router, Axios, and other required scripts:
   ```bash
   npm install
   ```

3. **Configure Environment Variables**:
   Create a file named `.env` in the `frontended/` root directory (next to `package.json`):
   ```env
   REACT_APP_API_BASE_URL=http://localhost:8080/api/v1
   ```
   *Note: This Base URL points to the local Spring Boot backend running on port 8080.*

4. **Start the local development server**:
   ```bash
   npm start
   ```
   * The application will compile and automatically launch in your default web browser.
   * If it doesn't open automatically, navigate to: **`http://localhost:3000`**

## 📡 API Connection Catalog

All HTTP requests to the backend are organized inside the `src/services/` directory and use the `REACT_APP_API_BASE_URL` defined in `.env`.

### 1. Authentication Services
*Code file: `src/services/authService.js`*

* **User Registration (`POST /auth/register`)**:
  * Registers a new account.
  * *Payload*: `{ "name", "email", "password", "location" }`
* **User Login (`POST /auth/login`)**:
  * Logs the user in and retrieves their JWT token.
  * *Payload*: `{ "email", "password" }`
  * *Response*: `{ "token", "userId", "name", "email" }` (stored in `localStorage` for future requests).

### 2. User Services (Protected)
*Code file: `src/services/userService.js`*

* **Get Profile (`GET /users/{id}`)**: Retrieves the logged-in user's profile details.
* **Update Profile (`PUT /users/{id}`)**: Updates user's name and location.
* **Delete Account (`DELETE /users/{id}`)**: Deletes account and wipes historical records.

### 3. Advisory Services (Protected)
*Code file: `src/services/recommendService.js`*

* **Crop Recommendation (`POST /recommend/crop`)**:
  * Recommends the optimal crop based on soil and weather metrics.
  * *Payload*: `{ "N", "P", "K", "temperature", "humidity", "ph", "rainfall" }`
* **Fertilizer Advisory (`POST /recommend/fertilizer`)**:
  * Recommends fertilizer based on crop and soil conditions.
  * *Payload*: `{ "cropName", "soilType", "nitrogen", "phosphorus", "potassium" }`
* **Irrigation Guidance (`POST /recommend/irrigation`)**:
  * Recommends daily water volumes and watering intervals.
  * *Payload*: `{ "cropName", "soilMoisture", "temperature", "growthStage" }`
* **Recommendation History (`GET /recommend/history/{userId}`)**:
  * Fetches all past reports generated by the user.

### 4. Disease Detection Services (Protected)
*Code file: `src/services/diseaseService.js`*

* **Detect Disease (`POST /disease/detect`)**:
  * Identifies plant disease from symptoms and suggests treatment/prevention.
  * *Payload*: `{ "cropName", "symptoms": [] }`
* **Supported Diseases List (`GET /disease/list`)**:
  * Fetches the catalog of all diagnosable crop diseases.

## 🔐 Authentication & JWT Injection

All protected API endpoints require an active JSON Web Token (JWT). 
1. Upon a successful login, the JWT string is saved to local storage as `sca_token`.
2. The service files dynamically retrieve this token and inject it into the HTTP headers of every request automatically using helper functions:
   ```javascript
   const getAuthHeaders = () => {
     const token = localStorage.getItem("sca_token");
     return {
       headers: {
         Authorization: `Bearer ${token}`
       }
     };
   };
   ```
