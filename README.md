## 🌱 Project Description

**Smart Crop Advisor** is a full-stack, AI-powered web application built to assist farmers and agricultural workers with smart, data-driven decisions. The system accepts field-level inputs from the user — such as soil nutrient levels, weather data, and crop symptoms — and returns intelligent recommendations generated by trained Machine Learning models.

The application is divided into three core modules:

- A **React.js Frontend** that the farmer or user interacts with directly through a web browser
- A **Java Spring Boot Backend** that acts as the brain of the system — handling all business logic, security, data storage, and coordination between modules
- A **Python AI/ML Module** that hosts trained Machine Learning models and processes prediction requests from the backend

All three modules run as independent services and communicate with each other over HTTP using REST APIs. The data is persisted in a **MySQL relational database** managed by the backend.

---

## 🚨 Problem We Are Solving

Agriculture is the backbone of millions of families, yet most farmers — especially small and marginal ones — make critical decisions about crops, fertilizers, and irrigation based on guesswork or outdated advice. This leads to:

- **Wrong crop selection** → poor yield, wasted effort
- **Overuse or underuse of fertilizers** → soil damage, financial loss
- **Undetected crop diseases** → entire harvests destroyed
- **Inefficient irrigation** → water wastage in water-scarce regions
- **No personalized advice** → generic suggestions that don't match local soil or climate

**Our solution:** A farmer types in their field data — soil nutrients, temperature, humidity, pH, and rainfall — and our system instantly returns AI-generated, personalized recommendations for which crop to grow, which fertilizer to use, how much to irrigate, and whether any disease risk exists. All powered by real ML models trained on verified agricultural datasets.

| Connection | Protocol | Format | Direction |
|---|---|---|---|
| Frontend ↔ Backend | HTTP/HTTPS | JSON | Both ways (request/response) |

| AI Module ↔ Models | In-memory | Python objects | Internal (loads `.pkl` files) |

## 🎨 Frontend Module — What We Built

> **Owned by: Anubhav Tripathi (Frontend Developer)**
> **Technology: React.js, HTML, CSS, JavaScript, Axios**
> **Runs at: http://localhost:5173**

The frontend is a **Single Page Application (SPA)** built with React.js. This means the entire application loads once in the browser and then React dynamically shows and hides components as the user navigates — without reloading the page. The user interacts only with this layer. They never directly call the backend or AI module — the frontend does that for them.

---

### 📄 Pages & What Each Does

#### `Home.jsx` — Landing Page
The first page a visitor sees. It introduces the Smart Crop Advisor project, explains what it does, and provides navigation buttons to Login or Register. It is a static informational page — no API calls.

#### `Login.jsx` — User Login Page
Displays a form with two fields: **Email** and **Password**. When the farmer submits, the frontend calls `authService.login()` which sends a `POST /api/v1/auth/login` request to the backend. On success, the backend returns a **JWT token** and the user's basic info. The frontend stores the JWT in `AuthContext` (React global state) and redirects to the Dashboard. If login fails, an error message is shown.

#### `Register.jsx` — New Account Registration
Form with fields: **Name, Email, Password, Location**. On submit, calls `authService.register()` → `POST /api/v1/auth/register`. On success, the user is redirected to Login.

#### `Dashboard.jsx` — Main Page After Login
This is the home page after a farmer logs in. It shows:
- A welcome message with the farmer's name
- Summary cards: total recommendations made, last recommendation date
- A table/list of the farmer's recommendation history (fetched from backend via `GET /api/v1/recommend/history/{userId}`)
- Quick action buttons to navigate to each advisory feature

#### `CropRecommend.jsx` — Crop Recommendation
This is one of the most important pages. It contains a form where the farmer enters:
- **Nitrogen (N)** value in soil
- **Phosphorus (P)** value in soil
- **Potassium (K)** value in soil
- **Temperature** (°C)
- **Humidity** (%)
- **pH** level of soil
- **Rainfall** (mm)

On submit, the frontend calls `POST /api/v1/recommend/crop` with this data. The backend processes it through the AI module and returns the recommended crop name, confidence score, alternative crops, and an advisory note. The result is displayed using the `RecommendationCard` component.

#### `FertilizerRecommend.jsx` — Fertilizer Recommendation
Similar form structure — takes crop name, soil type, and NPK values. Calls `POST /api/v1/recommend/fertilizer`. Returns fertilizer type (e.g., Urea, DAP, NPK blend) and application guidance.

#### `IrrigationAdvisory.jsx` — Irrigation Guidance
Takes crop type, soil moisture level, and temperature. Calls `POST /api/v1/recommend/irrigation`. Returns estimated water requirement per day and irrigation frequency recommendation.

#### `DiseaseDetect.jsx` — Disease Detection
Farmer selects their crop and describes visible symptoms (or selects from a symptom checklist). Calls `POST /api/v1/disease/detect`. Returns the most likely disease name, description, and suggested treatment/prevention measures.

#### `Profile.jsx` — User Profile
Shows the logged-in farmer's details. Allows editing name and location. Calls `PUT /api/v1/users/{id}` to update.

### 🔐 Authentication Flow in Frontend

1. User logs in → JWT token received from backend
2. JWT is stored in `AuthContext` (React context, in memory)
3. Axios is configured with an **interceptor** that automatically adds `Authorization: Bearer <token>` to every API request header
4. If a response returns `401 Unauthorized`, the interceptor automatically clears the token and redirects to Login
5. Protected routes (Dashboard, Recommend pages) check `AuthContext` — if no token, user is redirected to Login

### 🌐 How Frontend Calls the Backend

All API calls are centralized in the `services/` folder. No component directly calls Axios — they all use service functions. This makes it easy to update API URLs in one place.

```
services/
├── authService.js         → login(email, password), register(data), logout()
├── recommendService.js    → getCropRecommend(soilData), getFertilizer(data), getIrrigation(data)
├── diseaseService.js      → detectDisease(cropName, symptoms)
└── userService.js         → getProfile(userId), updateProfile(userId, data)
```

## ⚙️ Backend Module — What We Built

> **Owned by: Arpit Umrao (Backend Developer)**
> **Technology: Java 17, Spring Boot 3, Spring Security, JPA, Maven, MySQL**
> **Runs at: http://localhost:8080**

The backend is the **central coordinator** of the entire application. It receives all requests from the frontend, validates them, applies business logic, stores data in MySQL, calls the AI module when predictions are needed, and sends back structured responses.

The backend is built using the **MVC (Model-View-Controller)** pattern with a service layer in between — a standard professional Java architecture.

---

### 🏛️ Backend Layer Breakdown

```
HTTP Request from Frontend
        ↓
[Controller Layer]       ← Receives the request, reads URL/body/headers
        ↓
[Service Layer]          ← Applies business logic, validation, decisions
        ↓
[Repository Layer]       ← Queries MySQL database using JPA
        ↓
[MySQL Database]         ← Stores/retrieves data
        ↑
[AI Module Call]         ← Service calls Python AI module via RestTemplate (for prediction requests)
```

### 🎮 Controllers — The Entry Points of the Backend

Controllers are Java classes annotated with `@RestController`. Each method inside a controller is mapped to a specific URL and HTTP method using annotations like `@GetMapping`, `@PostMapping`, etc.

**`AuthController.java`**
- `POST /api/v1/auth/register` — Receives `{name, email, password, location}`, calls `AuthService.register()`, returns success or error message
- `POST /api/v1/auth/login` — Receives `{email, password}`, calls `AuthService.login()`, returns `{token, userId, name}` on success

**`RecommendController.java`**
- `POST /api/v1/recommend/crop` — Receives soil data, calls `RecommendService.getCropRecommendation()`, returns prediction result
- `POST /api/v1/recommend/fertilizer` — Receives crop + soil data, returns fertilizer recommendation
- `POST /api/v1/recommend/irrigation` — Receives crop + environmental data, returns irrigation advisory
- `GET /api/v1/recommend/history/{userId}` — Returns list of all past recommendations for a user

**`DiseaseController.java`**
- `POST /api/v1/disease/detect` — Receives crop name and symptoms, calls `DiseaseService.detect()`, returns disease result
- `GET /api/v1/disease/list` — Returns all supported diseases from the database

**`UserController.java`**
- `GET /api/v1/users/{id}` — Returns farmer profile data
- `PUT /api/v1/users/{id}` — Updates profile fields
- `DELETE /api/v1/users/{id}` — Deletes account

### 🧠 Service Layer — The Brain of the Backend

Services contain all the **business logic**. Controllers should not contain logic — they only receive requests and pass them to services. Services then decide what to do, call repositories for data, and call external services like the AI module.

**`AuthService.java`**
- `register()` — Checks if email already exists in DB. If not, hashes the password using BCrypt and saves the new user. Returns a success response.
- `login()` — Finds user by email. Compares the submitted password with the stored BCrypt hash. If valid, generates a JWT token using `JwtUtil` and returns it.

**`RecommendService.java`**
- `getCropRecommendation(soilData)` — Takes the soil data DTO, builds a JSON request, calls the Python AI module at `POST http://localhost:5000/predict/crop` using `RestTemplate`. Gets back the prediction result, saves the input and result to MySQL, and returns the result to the controller.
- Same pattern for fertilizer and irrigation.

**`DiseaseService.java`**
- `detect(cropName, symptoms)` — Calls AI module at `/predict/disease`, gets prediction, saves to DB, returns result.

**`UserService.java`**
- Standard CRUD operations for user profile management.

### 🗃️ Repository Layer — Talking to the Database

Repositories are interfaces that extend `JpaRepository`. Spring Data JPA automatically generates all SQL queries. We don't write `SELECT * FROM users WHERE email = ?` manually — JPA generates this from method names.

```java
// Example from UserRepository.java
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByEmail(String email);  // JPA auto-generates: SELECT * FROM users WHERE email = ?
    boolean existsByEmail(String email);        // JPA auto-generates: SELECT COUNT(*) FROM users WHERE email = ?
}
```

```java
// Example from RecommendationRepository.java
public interface RecommendationRepository extends JpaRepository<Recommendation, Long> {
    List<Recommendation> findByUserIdOrderByCreatedAtDesc(Long userId); // All recs for a user, newest first
}
```

### 🔒 Security — JWT Authentication

The backend uses **Spring Security with JWT (JSON Web Tokens)** to protect API endpoints. Here's how it works:

1. When a user logs in, the backend generates a JWT token containing the user's ID and email, signed with a secret key, valid for 24 hours.
2. Every subsequent API call from the frontend must include this token in the request header: `Authorization: Bearer <token>`
3. A **JWT Filter** (`JwtAuthFilter.java`) runs before every request. It reads the token, validates it, extracts the user's identity, and sets it in Spring's security context.
4. Routes like `/auth/register` and `/auth/login` are **public** (no token needed). All other routes are **protected** (token required).
5. If the token is invalid or expired, the request is rejected with a `401 Unauthorized` response.

### 🧪 Model Training Process (`train.py` and Jupyter Notebooks)

For each prediction task, the following pipeline was followed:

**Step 1 — Load and Explore Data**
```python
import pandas as pd
df = pd.read_csv('data/crop_recommendation.csv')
df.info()          # check column types and null values
df.describe()      # check distributions and ranges
df['label'].value_counts()  # check class balance
```

**Step 2 — Preprocess Data**
```python
from sklearn.preprocessing import LabelEncoder, StandardScaler

# Encode categorical target (crop name) into numbers
le = LabelEncoder()
df['label_encoded'] = le.fit_transform(df['label'])

# Separate features and target
X = df[['N', 'P', 'K', 'temperature', 'humidity', 'ph', 'rainfall']]
y = df['label_encoded']

# Scale features (so no single feature dominates)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
```

**Step 3 — Split into Train and Test Sets**
```python
from sklearn.model_selection import train_test_split
X_train, X_test, y_train, y_test = train_test_split(X_scaled, y, test_size=0.2, random_state=42)
# 80% data for training, 20% for testing model accuracy
```

**Step 4 — Train Multiple Models and Compare**
```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.svm import SVC
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score

models = {
    "Random Forest": RandomForestClassifier(n_estimators=100),
    "SVM": SVC(kernel='rbf', probability=True),
    "KNN": KNeighborsClassifier(n_neighbors=5)
}

for name, model in models.items():
    model.fit(X_train, y_train)
    acc = accuracy_score(y_test, model.predict(X_test))
    print(f"{name}: {acc * 100:.2f}%")
```

**Step 5 — Select Best Model and Save It**
```python
import joblib
# Random Forest gave highest accuracy (~99%) → chosen as final model
best_model = RandomForestClassifier(n_estimators=100)
best_model.fit(X_scaled, y)
joblib.dump(best_model, 'models/crop_model.pkl')
joblib.dump(le, 'models/label_encoders.pkl')
joblib.dump(scaler, 'models/scaler.pkl')
```

The same training pipeline is followed for the fertilizer and disease models, with their respective datasets and target labels.

### 🔮 Prediction Logic (`predict.py`)

When the FastAPI server receives a request, `predict.py` is called:

```python
import joblib, numpy as np

# Load pre-trained models once at startup (not on every request — for speed)
crop_model = joblib.load('models/crop_model.pkl')
label_encoder = joblib.load('models/label_encoders.pkl')
scaler = joblib.load('models/scaler.pkl')

def predict_crop(N, P, K, temperature, humidity, ph, rainfall):
    input_data = np.array([[N, P, K, temperature, humidity, ph, rainfall]])
    input_scaled = scaler.transform(input_data)
    prediction_encoded = crop_model.predict(input_scaled)
    predicted_crop = label_encoder.inverse_transform(prediction_encoded)[0]
    confidence = max(crop_model.predict_proba(input_scaled)[0]) * 100
    return predicted_crop, round(confidence, 2)
```

### 🌐 API Server (`app.py`)

The FastAPI server exposes three prediction endpoints that the Java backend calls:

```python
from flask import Flask, request, jsonify
from predict import predict_crop, predict_fertilizer, predict_disease

app = Flask(__name__)

@app.route('/predict/crop', methods=['POST'])
def crop_prediction():
    data = request.json
    crop, confidence = predict_crop(
        data['N'], data['P'], data['K'],
        data['temperature'], data['humidity'],
        data['ph'], data['rainfall']
    )
    return jsonify({"prediction": crop, "confidence": confidence})

@app.route('/predict/fertilizer', methods=['POST'])
def fertilizer_prediction():
    data = request.json
    fertilizer = predict_fertilizer(data)
    return jsonify({"prediction": fertilizer})

@app.route('/predict/disease', methods=['POST'])
def disease_prediction():
    data = request.json
    disease = predict_disease(data['crop'], data['symptoms'])
    return jsonify({"prediction": disease})

if __name__ == '__main__':
    app.run(port=5000, debug=True)
```

## 🔗 How Everything Connects — End-to-End Flow

Here is a complete walkthrough of what happens when a farmer submits the Crop Recommendation form:

```
1. 👨‍🌾 Farmer fills in the form on CropRecommend.jsx (React)
        ↓
2. 📤 React calls recommendService.getCropRecommend(formData)
        ↓
3. 🌐 Axios sends:
        POST http://localhost:8080/api/v1/recommend/crop
        Headers: { Authorization: Bearer <JWT> }
        Body: { N:90, P:42, K:43, temperature:25.5, humidity:80, ph:6.5, rainfall:202.9 }
        ↓
4. 🔐 Spring Security's JWT Filter intercepts the request
        → Reads the Bearer token from header
        → Validates signature and expiry
        → Extracts user ID, sets in SecurityContext
        → Passes request through to controller
        ↓
5. 🎮 RecommendController.java receives the request
        → Parses body into CropRecommendRequest DTO
        → Calls RecommendService.getCropRecommendation(dto, userId)
        ↓
6. 🧠 RecommendService.java takes over:
        a. Saves the soil input to MySQL (soil_inputs table)
        b. Calls AI module:
              POST http://localhost:5000/predict/crop
              Body: { N:90, P:42, K:43, temperature:25.5, humidity:80, ph:6.5, rainfall:202.9 }
        ↓
7. 🤖 Python FastAPI server receives the call at /predict/crop
        → predict.py loads crop_model.pkl and scaler.pkl
        → Scales the input data using StandardScaler
        → Runs model.predict() → gets "Rice" (encoded)
        → Runs label_encoder.inverse_transform() → gets "Rice" (string)
        → Gets confidence: max(model.predict_proba()) = 94.7%
        → Returns: { "prediction": "Rice", "confidence": 94.7 }
        ↓
8. 🧠 RecommendService.java receives AI response
        → Saves recommendation to MySQL (recommendations table)
        → Builds RecommendationResponse DTO with crop, confidence, note, alternatives
        → Returns to controller
        ↓
9. 🎮 RecommendController.java returns HTTP 200 OK with JSON response body
        ↓
10. 🌐 Axios receives the response in React
        ↓
11. 🎨 RecommendationCard.jsx renders the result on screen:
        → Shows "Recommended Crop: RICE"
        → Shows confidence badge: "94.7% confidence"
        → Shows advisory note
        → Shows alternative crops: Maize, Jute
```

Every single step above happens in under 2–3 seconds.
